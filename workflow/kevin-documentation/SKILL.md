# Kevin's Documentation Skill

Patterns for end-of-session documentation, creating project instruction files, and extracting reusable patterns.

## When to Use Which Reference

| Task | Reference File | Key Approach |
|------|----------------|-------------|
| **End-of-session docs, session summaries** | `references/session-docs.md` | Update NOW.md, DEVLOG.md, breadcrumbs. Archive plans. Don't over-document. |
| **Creating a new CLAUDE.md** | `references/claude-md-creation.md` | Keep minimal. Ask followups. Under 200 lines. Reasoning over rules. |
| **Extracting a reusable pattern** | `references/pattern-extraction.md` | After a productive session, capture the repeatable pattern as a skill or guide. |

## Documentation Philosophy

- **Useful density over completeness.** Enough detail to reconstruct reasoning, not a transcript.
- **Separate audiences:** KEVIN-TIPS for humans, skills and CLAUDE.md for Claude, guides for processes.
- **Don't document mid-stream.** Focus on the work. Document at the end or at natural breakpoints.
- **Single source of truth.** If it's documented in a guide, don't duplicate it in the CLAUDE.md.
